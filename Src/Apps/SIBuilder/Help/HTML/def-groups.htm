About File Groups

File groups are used to group files together that are to be installed according to the same set of rules.

A file group has the following attributes:

  • A name – this is displayed by SIBuilder and used to identify the group.
  • An installation path – this is the path on which all files in the group will be installed.
  • A rule governing how the folder is to be removed. This can be any of:
    • the folder is never removed;
    • the folder is only removed by the uninstaller if it is empty after files have been uninstalled;
    • the folder and all its sub-folder and files will be removed (the file removal options are ignored) by the uninstaller.
  • A rule by which the group's files will be deleted. This can be any of:
    • all files in the folder will be deleted by the uninstaller;
    • only those files that were installed in the folder will be deleted by the uninstaller;
    • no files will be deleted;
    • the files installed in the folder will be deleted by the installer (after they have been used) so these files are not available for use by the uninstaller – the option is designed for programs to be run from the installer.
  • The set of operating systems under which the group will be installed. If the operating system on which the program is being installed is not in the set of those valid for the group then the group is not installed.

Therefore files must only be placed in the same group if:

  • they are to be installed in the same folder;
  • they are all to be treated identically when being uninstalled;
  • they are all to be installed on the same set of operating systems.
