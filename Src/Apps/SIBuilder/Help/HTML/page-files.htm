Files Page

The files page is where files to be installed are assembled. Files are collected together in Groups.

The page contains a tree view of the project's files – groups are represented by folder icons and the name of the group is displayed. Files contained within groups are displayed under groups. Each file is represented by an icon according to its type along with the full path to the source file.

By default a blank project comes with just one predefined group – "Main". This group will install files into the installation folder specified on the Project Details page. The "Main" group can be changed or deleted as required.

Groups are updated by using the buttons on the right of the page, or by the equivalent items on group tree view's popup menu. The following commands are available:

Add Group Adds a new group. The Edit Group Properties dialog box is displayed where the details of the new group can be entered. Path macros can be included in the group's installation path.
Edit Group Edits the selected group, again using the Edit Group Properties dialog box.
Delete Group Deletes the selected group, after seeking confirmation.
Add Files Adds one or more files to the currently selected group. Displays the Add Files dialog box in which multiple files can be selected. The files added have default properties assigned to them (see Edit File below). The properties can be changed by editing the file.
Edit File Edits the file's installation properties using the Edit File Install Properties dialog box. The options which can be configured are those for overwriting existing files and whether file is registered in the Windows registry.
Remove File Removes the currently selected file from the group without prompting.
File Properties Displays the File Properties dialog box which shows modification date, file attributes, modification date and size, along with any version information.

Files can also be added to a group by dragging and dropping files from explorer and dropping them over a group in the tree view. If no group is under the cursor when the files are dropped a message is displayed to say the drop has been unsuccessful. Note: Dropping any file outside the treeview causes SIBuilder to attempt to open it as a project file.

An empty folder can be created as part of the installation (say a folder for a users work files). To do this create a group that contains no files.

When a file or group is selected a pop-up information window may appear that displays key information about the selected item. These pop-up windows can be prevented from appearing using the Options dialog box.

If there are any problems with any of the files on the page then its icon is replaced by a symbol and a clickable prompt appears under the page's buttons which leads to help on what may be wrong.
