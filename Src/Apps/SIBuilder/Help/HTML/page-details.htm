Project Details Page

The project details page is where essential basic general information about the project is provided.

This page should always be completed first when starting a new project – in fact starting a new project or opening an existing one always activates this page. It is not possible to leave the page until all required items have been provided.

The page is split into four sections. The purpose and use of each section is described below.

Project Identification

The project ID identifies your project to SIBuilder and must be provided. Enter a brief, unique, identifier for the project in the Project ID edit box. The ID must be a valid Pascal identifier. Examples of appropriate IDs include "MyProject" and "SIBuilder1".

A longer, descriptive, name for the project must also be provided. This name is used to identify the project to human readers, for example, by the Add/Remove programs Control Panel applet. This name is compulsory and should be entered in the Project Name edit box.

Installation Path

The Installation Path is the main (or root) folder on the target system in which the project is to be installed. A path can be entered either by typing it into the edit box or by clicking the button at the right end of the edit box and choosing a path from the Choose folder dialog box that is then displayed. The installation path can contain any path macro except [Install Files].

Whether the installation path may be edited by the person installing the program is specified by checking or clearing the Editing of installation path permitted? check box. If this check box is checked then the Ask user for path on installation? check box becomes available. Checking it causes the install program to pause and ask the user to enter the required path. If this box is left clear then the only way for the user to change the installation path is to pass it as a command line parameter to the install program.

Install Program Info

The name to be used for the install file is entered in the Install program name edit box. This is Install.exe by default. The install file is created in the same folder as the project file, so no path should be specified.

Determine whether SIBuilder will create an uninstall program by checking or clearing the Create uninstall program? check box. If created, the uninstall program is always named UnInstall.exe.

Previous Installations

The installer can require other applications to be uninstalled before the installation continues. The identifiers of the programs to be uninstalled are entered in the Uninstall projects(s) edit box – multiple identifiers are separated by semi-colons. As an alternative a list of identifiers can be specified by clicking the Edit button embedded in the edit control and using the Edit Projects to be Uninstalled dialog box to enter the identifiers. The identifiers to specify are those that are used to identify a program to the WIndows Add/Remove programs control panel applet in the Windows registry.

The install program can also detect if a previous (SIBuilder generated) installation already exists in the folder it is being installed into. If this is the case the installer can uninstall the previous installation before performing its own installation. This is the default behaviour and is required by some project options. The installer can be prevented from uninstalling previous installations by clearing the Uninstall previous installations in same folder check box.

When all entries have been made it is possible to move to other pages. If there any missing or invalid entries on this page then a message will be displayed and the page cannot be changed.
