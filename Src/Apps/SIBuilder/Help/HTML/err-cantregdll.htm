Shared DLL Registration Error

It is an error for a file to be flagged as a shared DLL when the project does not permit uninstallation of pre-existing programs in the same folder.

This is because overwriting an existing project without properly uninstalling it may cause a previously counted shared DLL to be overwritten without its usage count being decreased. This would cause the count to get out if sync with the number of programs actually using the DLL.

Solutions

  1. edit the file to remove the Shared DLL flag in the Edit File Install Properties dialog box (accessed via the Edit File button on the Files page;
  2. go to the Project Details page and check the Uninstall previous installations in same folder check box.
