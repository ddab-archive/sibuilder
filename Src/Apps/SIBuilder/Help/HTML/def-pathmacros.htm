About Path Macros

Path Macros enable paths to be specified in a system independent way. Path macros are predefined names that can appear at the start of a file path or file name specification. The macros that are supported are:

  • [Install Path] – this is replaced by the project's installation path (which may itself contain any of the other macros).
  • [Program Files] – this is replaced by the Program Files folder (e.g. C:\Program Files).
  • [Common Files] – this is replaced by the Common Files folder (e.g. C:\Program Files\Common Files).
  • [Windows] – represents the Windows folder on the target system (e.g. C:\Windows).
  • [System] – represents the System folder on the target system (e.g. C:\Windows\System).
  • [Temp] – represents the user's temporary folder on the target system (e.g. C:\Windows\Temp).
  • [Desktop] – represents the user's desktop directory on the target system (e.g. C:\Documents and Settings\\Desktop).
  • [Programs Menu] – represents the user's start menu program folder on the target system (e.g C:\Documents and Settings\\Start Menu\Programs).
  • [ScratchPad] – represents a scratch pad folder available during installation only. This is a sub folder of [Temp] that has name defined by the installer and that is deleted once the installer completes. This path macro is used by the installer and users are recommended not to use it.

When paths are entered in SIBuilder that match any of the macros, SIBuilder replaces the path with the macro.

Path macros are expanded by the installation program. This means that you can be certain that files can be installed in any of these folders on the target system even if the actual folders are different to the system where SIBuilder created the project. If the project permits it, the person installing the program can specify their own value for the [Install Path] macro.

Note: You should not install any files into [Temp] or [ScratchPad] that must be available after the installer completes. The user may empty the [Temp] folder at any time and the installer will try to remove [ScratchPad] when it completes.
