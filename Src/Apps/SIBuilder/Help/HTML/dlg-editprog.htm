Edit Program Details Dialog Box

This dialog box enables the editing details of a program that is to be run by the installer, uninstaller or both. The dialog is displayed when either the Add Program or Edit Program button on the Programs Page is clicked.

The information to be entered is divided into two sections, with corresponding group boxes on the dialog: Program Details and Command Line Parameters. Each of these sections is discussed below.

Program Details

The Program Details group box is where the name of the program to be run is specified, along with when it is to be run and whether it should be deleted after execution. The controls are explained below.

The program to be run is specified in the Program combo box. All executable programs currently included in the project are listed in the combo box's drop down list, so to choose an included program simply select it from the list. Alternatively an external program on the target system can be specified. To do this enter the program's full path on the target system. Path macros can be used and the combo box lists them in the drop down list to ease entry. An external program can also be chosen from the local computer by selecting the Browse for file item from the combo list and choosing the file from the resulting Choose Program File dialog box. Any standard paths in the chosen file are converted to path macros.

The Execute on drop down list should be used to determine when the selected program is to be run. The options are:

Installation only The program is run by the installer after all files have been installed.
Uninstallation only The program is run by the uninstaller before any files are deleted.
Installation and Uninstallation The program is run by the installer after installing files and by the uninstaller before deleting any files.

Do not specifiy a program to be run by the uninstaller if either:

  1. the program is being installed by the installer in a temporary group (since the program will be deleted by the installer and therefore be unavailable to the uninstaller), or
  2. if no uninstall program is included in the project.

If either of these mistakes are made then a message (preceeded by a symbol) will be displayed alongside the Execute on combo box. Click the message's hot link to read about possible causes of problems.

Command Line Parameters

The controls in this group box are used to enter details of the commands to pass to the program described in the Program Details group box. Each parameter or switch is entered separately and is displayed on its own line in the All parameters list box.

Zero or more command line parameters can be passed to the program. Parameters that contain spaces will be quoted before being passed to the program, so there is no need to include quotes. Path macros may be embedded in parameters and will be expanded by the installer. The parameters will be passed in the order they are specified.

To add a parameter enter it in the Parameter combo box and click the Add button. The parameter now appears in the All parameters list box. (File parameters may include path macros).

To edit an existing parameter, select it in the All parameters list box and click the Edit button (or double click the parameter). The parameter is copied to the Parameters combo box and can be edited there. The Add button's caption changes to Update and this is clicked to accept the edit and to update the selected parameter.

Delete a parameter by selecting it in the All parameters list box and clicking the Delete button.

Parameters can be re-ordered by selecting the parameter to be moved and changing its position in the list using the and buttons. (Keyboard alternative: CTRL+UP and CTRL+DOWN).

Ensure that no temporary file is included in a parameter of a program that is to be run by the uninstaller since the file will have been deleted by the installer. If this is the case then an error message (preceeded by a symbol) will be displayed above the parameters list. Click the hot link contained in the message to get more information.
