v2 File Conversion Comment:
Ignored "Register Only" Property for COM Server

SIBuilder 2 provided a "register only" property for in-process COM servers that enabled a COM server to be registered by the installer but ignored by the uninstaller. It was left to the user to ensure that the server wasn't deleted by the uninstaller!

SIBuilder 3 does not support this property. This conversion comment indicates that such a property was encountered in the project file and has been ignored by SIBuilder. The uninstaller will automatically unregister any COM server that was registered by the program before deleting the file. If the file is not deleted then it is not unregistered.

The server file's entry on the Files page will show that the installer will register it as before.

It is recommended that all In-process COM servers are flagged to be registered as shared DLLs as well as being registered as COM servers. This way the server will only be deleted once it is no longer required by programs. To do this select the file on the Files page and edit it using the Edit File button to display the Edit Install File Properties dialog box.
