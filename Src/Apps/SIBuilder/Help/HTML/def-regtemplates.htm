About Registry Templates

Registry templates are designed to make it easy to enter commonly used registry keys in the project's registry tree.

Each template provides a registry path. Some of these paths are fixed registry keys (e.g. the "run once" key in HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\RunOnce). However other keys have variable parts to them, such as the key used to associate an extension with a file type or application KEY_CLASSES_ROOT\.ext) where the extension .ext needs to be replaced by the required value every time the key is used.

Registry templates contain "macros" wherever a such variable item occurs in a key. These act as placeholders for the required value. Macros are enclosed in brackets like this: [MacroName]. So the template for associating an extension with a file type or application is recorded as KEY_CLASSES_ROOT\[FileExt]. When a template is used each macro can be replaced by a value – and the macro remembers its recent values and its current value. Most of the predefined templates use macros.

When a template is presented to the user the most recent value for each macro (if any) is used. The Choose Registry Template dialog lets the user replace the value with any recent value or provide a new one. When a template is added to the registry the most recently used value is subsituted for each macro.
