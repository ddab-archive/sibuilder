v2 File Conversion Comment:
Program Moved to Temporary Install Group

Programs specified to be run by the installer under SIBuilder 2 could be set to be deleted after execution. These were the only files the the installer was able to delete (other than the license file), so, for example, any documents displayed by the program had to remain on the system. The programs to be deleted were specified on the Programs page.

With SIBuilder 3 the facility has been extended so that the installer can now delete any installed file. To acheive this the files to be deleted are included in a new deletable group on the Files page. This means that programs themselves are no longer marked as deletable.

When loading a SIBuilder 2 project file that specifies an installed program for deletion we need to use the new method to ensure they are actually deleted. This is done by creating a special deletable group to install the program file from and moving it to there from its original group.

Note 1: if the program accesses any other files (possibly specified as command line parameters) and those files are not required elsewhere in the installed program then the user may wish to also move those files to a temporary group. Such files are not moved automatically by SIBuilder 3 since there is no way of knowing if the files are needed elsewhere.

Note 2: For security reasons SIBuilder no longer allows external programs (i.e. those not included on the install project) to be deleted after being run.
