Build Installer Page

This final page is used to build the install program using all information entered on the previous pages.

Before the project is built the compression method to be used compress the install files is selected using the Compression drop down list.

The project can be built by clicking the Build button. The page is then disabled and the build progress panel is raised up while the build takes place. The build, the progress of which is displayed on the raised panel, has several key stages:

  1. If you have not saved your project then you will be asked to do so and the Save Project dialog box will be displayed. If the project has already been saved it is re-saved to update the project file before building takes place.
  2. The project is then verified to check for any errors, such as missing files.
  3. The data that will be embedded in the install program is then assembled and compressed as required. This is the longest phase of the build.
  4. The install program is finally created. The install file is given the name specified on the Project Details page and is stored in the same folder as the project file.

If any errors are encountered a message is displayed identifying the error. If the project is successfully built then a confirmation message is displayed.

Once the project has been saved a report of the files contained in the project can be produced. Generate the report, which is stored in the same folder as the project file, by clicking the Report button. The report is produced either in plain text or HTML, according to the settings made in the Options dialog box. The report can be viewed in your default text editor or browser (depending on type) by clicking the View Report button.

That's it! The install program is created and ready for distribution. However, you are advised to test the installation before distributing it. See Using Install and UnInstall for further information.
