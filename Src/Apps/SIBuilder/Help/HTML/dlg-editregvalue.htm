Edit Registry Value Dialog Box

This dialog box is displayed when the Add Data Item or Edit Data Item buttons from the Registry Page are clicked. It is used to enter new values under registry keys or to edit existing values.

Name

The name of the data item is entered in the Name edit box. The data item's name must be unique for the key it belongs to. The empty string may be entered to denote the default data item for the key.

Data Type

The available types are Integer, Binary, String and Expand String. Choose the required type by selecting the appropriate radio button. If you choose one of the string types and require the installer to expand path macros then you should check the Expand Path Macros check box.

Value

The data item's value is entered in the Value text box. The format of the value must be appropriate for the data type selected in the Data type radio group. Permitted values are:

Integer: Enter a decimal whole number or a hexadecimal number preceded by a $ symbol. The value must be such that it can be stored as a Long Integer or DWORD.
Binary:
Enter the binary data as space separated hexadecimal bytes. E.g.
FF EE 4C 43 6A 00
represents six bytes of binary data.
String: Enter the string as simple text. Do not press return unless a CR LF pair is required in the string. If Expand Path Macros is checked then any path macro entered in the text box will be expanded when the installation program is run.
Expand String: Entry rules are the same as for string above. The expand string type also support path maco expansion if required. An expanded string also contains unexpanded references to environment variables (delimited by % characters). See the Win 32 programmers reference for further information. Do not embed path macros within environment variables as the behaviour is not defined.

Alternative methods to enter values

If there is text data on the clipboard it can be pasted using the Paste data from clipboard button. The data is displayed in the Value text box and the String radio button will be selected. If there is no string data on the clipboard the Paste data from clipboard button is disabled.

Binary data can be read from a file. To do this click the Read data from file button and choose the required file from the resulting Open Binary Data File dialog box. The contents of the file will be displayed in Hex notation in the Value text box and the Binary Data type radio button will be selected. Note: SIBuilder does not permit files of more than 2Kb to be stored in the registry.

If the String or Expanded String radio buttons are selected then a file name can be inserted by clicking the Insert File Name button and choosing a file from the resulting Insert File Name dialog box.

If the Expand Path Macros check box is checked and either the String or Expanded String data types are selected then a path macro can be inserted at the current insertion point in the Value editor by clicking the Insert Path Macro button to drop down a menu of avaialable path macros and choosing the required macro from it.
