v2 File Conversion Comment:
Ignored  External  In-Process COM Server

SIBuilder 3 does not support registration and unregistration of COM server DLLs that are not included in the installation project. This facility was provided in SIBuilder 2.

Therefore, when SIBuilder 3 encounters a section in a SIBuilder 2 file that specifies registration of such a server, the entry has to be ignored.

If it is important that a non-project COM server is registered then an external program can be developed to achieve this. Such a program can be included in the project and run by the installer (using the Programs page).
