v2 File Conversion Comment:
License DLL Registered a COM Server

SIBuilder uses a DLL &ndash SITLicenseDlg.dll – to display any license required by the installer. SIBuilder 2 enabled the user to require that the DLL was retained – ignored by the uninstaller – and registered as a COM server.

This facility is not directly available in SIBuilder 3 so the project file has been modified to ensure that the DLL is separately installed and is flagged to be registered as a COM server. The program will have created a new group, "SIP2 CONVERTER: License DLL", that can be viewed on the Files Page.

SIBuilder 3 provides alternatives that were not available in SIBuilder 2. See the License Dialog DLL Retained Without Deletion topic for further information.
