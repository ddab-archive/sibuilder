File Errors

SIBuilder has detected one or more problems on the Files page. The offending files are marked with the symbol in the tree view. The likely causes of the problems, and possible solutions, are listed below:

  1. The source for the given file does not exist on the host system.
    Solution: The entry should be deleted and possibly replaced with the correct path to the file.
  2. The file is flagged as a shared DLL but the project does not permit uninstallation of pre-existing programs in the same folder. This is problematic since overwriting an existing project may cause a previously counted shared DLL to be overwritten without its usage count being decreased. This will cause the count to get out if sync with the number of programs actually using the DLL.
    Solutions:
    1. edit the file to remove the Shared DLL flag;
    2. go to the Project Details page and check the Uninstall previous installations in same folder check box.
